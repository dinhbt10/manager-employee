MH2: Màn hình thông báo thành công

Logic MH quản lý request

/==================

Chức năng **Quản lý request** cho phép người dùng tạo và quản lý các yêu cầu (request) nhằm:

- Thêm mới nhân viên  
- Cấp quyền sử dụng chức năng cho nhân viên  
- Điều chỉnh quyền cho một hoặc nhiều nhân viên

Hệ thống áp dụng cơ chế **request – approval**, đảm bảo:

- Người dùng không thao tác trực tiếp lên dữ liệu chính  
- Mọi thay đổi phải thông qua phê duyệt của Admin

---

## **3.2. Các chức năng chính**

### **3.2.1. Tạo mới request**

**Mô tả:**

Người dùng tạo request để:

- Thêm mới nhân viên  
- Hoặc cấp thêm quyền chức năng cho nhân viên

---

**Thông tin nhập:**


| Trường            | Mô tả                     |
| ----------------- | ------------------------- |
| Tiêu đề           | Ví dụ: Thêm mới nhân viên |
| Nhận xét          | Ghi chú thêm              |
| Mã nhân viên      | Ví dụ: NV01               |
| Họ tên            |                           |
| Email             |                           |
| Phòng ban         |                           |
| Nhóm              |                           |
| Chức danh         |                           |
| Role              |                           |
| Chức năng cần cấp | Ví dụ: THEMNV             |


---

**Xử lý hệ thống:**

- Lưu vào bảng requestcongviec  
- Lưu chi tiết vào requesttaonhanvien  
- Trạng thái ban đầu: **New (Lưu nháp)** hoặc **Waiting (Chờ duyệt)**

---

### **3.2.2. Chỉnh sửa request**

**Mô tả:**

- Cho phép chỉnh sửa request khi đang ở trạng thái:  
  - Lưu nháp (Draft)  
  - Hoặc Chờ duyệt (Pending)

---

**Quy tắc:**

- Không được chỉnh sửa khi:  
  - Đã **Phê duyệt (Approved)**  
  - Đã **Từ chối (Rejected)**

---

**Xử lý:**

- Update dữ liệu trong:  
  - requestcongviec  
  - requesttaonhanvien

---

### **3.2.3. Duyệt request**

**Mô tả:**

Admin thực hiện duyệt request

---

**Luồng xử lý:**

1. Admin chọn request
2. Click **Phê duyệt**
3. Hệ thống:

---

### **🔥 Xử lý nghiệp vụ quan trọng:**

Ví dụ:

Request:  
 Tiêu đề: Thêm mới nhân viên  
 Mã NV: NV01  
 Chức năng: THEMNV

👉 Sau khi **Approved**:

Hệ thống thực hiện:

---

### **Trường hợp 1: Nhân viên chưa tồn tại**

- Insert vào nhanvien  
- Insert vào taikhoan

---

### **Trường hợp 2: Nhân viên đã tồn tại**

- Không tạo mới nhân viên  
- Chỉ cấp quyền

---

### **Cấp quyền:**

## Insert vào nhanvienchucnang:

 nhanvienid  NV01  
 chucnangid  THEMNV  

👉 Kết quả:

Nhân viên NV01 từ **không có quyền**  
 → trở thành **có quyền sử dụng chức năng “Thêm mới nhân viên”**

---

### **3.2.4. Từ chối request**

**Mô tả:**

Admin từ chối request nếu không hợp lệ

---

**Xử lý:**

- Update:

trangthai  rejected

- Không thay đổi dữ liệu hệ thống

## **3.3. Trạng thái request**


| Trạng thái | Ý nghĩa   |
| ---------- | --------- |

            → Rejected
